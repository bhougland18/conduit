## Recent Work

`conduit-introspection` now owns pure workflow and contract introspection:

- crate added at [crates/conduit-introspection/src/lib.rs](/home/ben/code/conduit/crates/conduit-introspection/src/lib.rs)
- workspace membership and dependency aliases added in [Cargo.toml](/home/ben/code/conduit/Cargo.toml)
- `introspect_workflow` validates workflow contracts and capabilities before projection
- `WorkflowIntrospection` exposes workflow id, node views, and edge views
- node views include ports, execution mode, enforcement level, determinism, retry disposition, and effect capabilities
- port views include direction, schema, and port capability direction
- edge views include source, target, capacity, and source/target schemas
- `EnforcementLevel` reports native nodes as advisory and WASM/process nodes as strict

The implementation uses a separate crate rather than `conduit-core::introspection`
because `conduit-contract` already depends on `conduit-core`; placing
contract-aware introspection in core would create a dependency cycle.
